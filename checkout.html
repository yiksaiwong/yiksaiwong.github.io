TOMO Checkout Redesign



Overview

Scrolling for unique products is easy on Touch of Modern (TOMO), an e-commerce website for modern men. However, customers are not completing their checkout process, leaving us with a cart abandonment rate of 67.83%.

The goal of this project was to redesign the checkout flow to reduce cart abandonment rate.

Responsibilities: User Research, Prototyping, Interaction Design, Visual Design, User Testing, Quality Assurance Testing

01. Research & Analysis

üòü Problems with the Current Checkout

It has been a few years since the checkout flow has been redesigned. Since then, we‚Äôve continuously added new features to the existing checkout despite various limitations. We analyzed our current flow and discovered the following pain points:

  • Screens have unique, inconsistent layout and styling
  • Lots of back and forth between steps (more evident on mobile web)
  • Lacks consideration for all use cases (e.g. Users cannot use fast checkout to purchase age-restricted items or select shipping speed)
  • Incomplete typing and error checking system
Current Checkout User Flow

‚ú® Opportunities to Improve

In order to improve the checkout process, we planned to implement the following changes after doing competitive analysis and checkout reassessment:

  • Create a more intuitive and linear experience
  • Make the checkout flow faster
  • Remove restrictions for fast checkout

üß≠ Explorations

Changing the Overall Layout

One key pain point of the current flow was that it requires customers to continuously reload screens to advance to the next section. This issue was especially frustrating for mobile users who had to do a lot of scrolling on top of the reloading. To solve this issue, we decided on a single page flow to avoid reload time and to provide consistent layout and styling for all customers. We also added indicators for each checkout step to create a smooth and intuitive experience.

Iterations for Overall Layout

Improve Gifting Option and Apply Store Credit

The current flow created a lot of back and forth between steps when selecting gifting options and applying store credit. It was confusing where and when users can edit their information. This inconsistency was improved through the single page checkout.

Improve Shipping and Billing Sections

Similarly, it took a lot of effort for customers to add new shipping or billing info. Selecting an existing shipping/billing is on a modal but adding a new shipping/billing requires a page reload. The single page checkout helped avoid page reload, thus expediting checkout flow. Additionally, we required users to type unnecessary information such as retyping their billing address even if it‚Äôs the same as their shipping address. These unneeded steps were removed to also make the flow faster.

Add a Typing and Error Checking System

Upon further inspection, checkout time tended to increase when customers inputted their information incorrectly. To solve this, we incorporated an address autocomplete feature, city and state autofill feature (for customers who skip address autocomplete), address verification feature, and an error checking system. The new error checking system applied to the rest of the site as well to make it a consistent experience for customers.

üîé User Testing

To validate our designs, we conducted 12 remote usability testing sessions with potential and existing customers. 6 were conducted on desktop web and 6 were conducted on mobile web. Each participant was asked to complete a set of common tasks and scenarios for online shoppers.

After reviewing the testing results with the stakeholders, the users had no major difficulties going through the checkout flow. However, there were a few hiccups and confusions. Thus, we improved the following features based on user feedback:

Autofill City and State

Customers who did not use Google address recommendation preferred filling out the Zip Code first and having it autofill City and State to save time.

Rewrite Copy

While all participants were able to successfully complete the checkout flow, there was some confusion around the copy. Some were confused why the ‚ÄòCard & Other Options‚Äô section led to the shipping section first instead of the billing section. Others were confused why we asked for a phone number in the shipping section. Due to this, we revisited the copy again to make sure all steps were clear.

03. Quality Assurance

üìù User Acceptance Test

To ensure quality assurance, the design team created an initial draft of the user acceptance test plan (UAT) to list out all the functionalities and use cases. The engineering team reviewed it for clarifications before editing as needed.

After the first test build was out, the designers went through the UAT to list out any UX and UI issues. A few rounds of testing is currently being done to ensure everything works smoothly.
