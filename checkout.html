TOMO Checkout Redesign

It is easy to spend hours scrolling for unique, cutting edge products on Touch of Modern (TOMO), an e-commerce website for modern men. Unfortunately, while products are being added to customers‚Äô carts, there is a cart abandonment rate of 67.83%. To optimize our checkout process, the goal of this project was to redesign the existing checkout flow to identify usability issues to reduce cart abandonment rate and checkout time for customers.

Role: UX Designer (User Research, Prototyping, Visual Design, Interactions, Usability Testing)

01. Research + Analysis

üòê Challenges

It has been a few years since our checkout flow has been redesigned. Throughout those years, we‚Äôve continuously added new features to the existing checkout despite various limitations. Due to that, the current checkout experience is just not the most optimal. After reassessing our current checkout flow with another designer, we discovered the following pain points:

  • Screens have unique, inconsistent layout and styling
  • Lots of back and forth between steps (more evident on mobile web)
  • Lacks consideration for all use cases (e.g. Users cannot use fast checkout to purchase age-restricted items or select shipping speed)
  • Incomplete typing and error checking system
Current Checkout User Flow

üéØ Goals

After doing an extensive study on competitors, it was clear that we had a lot of technical debt to pay. Through the findings of the study and the checkout reassessment, we identified the following main goals:

  • Make the checkout flow faster
  • Create a more intuitive and linear experience
  • Remove restrictions for fast checkout

üîó User Flow

After determining our goals, we created a user flow outlining the main pages and steps in order to get a clearer picture of all the different use cases. We also identified the optional and required steps needed to checkout.

02. Design + User Testing

üß≠ Explorations

Overall Layout

One key pain point of the current flow is that it required a lot of back and forth; users must continuously reload screens to advance to the next section. This issue was especially frustrating for mobile users who had to do a lot of scrolling on top of the reloading. To solve this issue, we decided on a single page flow to avoid reload time. This single page flow will work for all use cases and provide consistent styling for every customer. We also added indicators for the user‚Äôs current step and for optional steps to create a smooth and intuitive experience.

Iterations for Overall Layout
Improve Gifting Option and Apply Store Credit

The current flow created a lot of back and forth between steps when selecting gifting options and applying store credit. It was confusing where and when users can edit their information. This inconsistency was improved through the single page checkout.

Improve Shipping and Billing Sections

Similarly, it took a lot of effort for customers to add new shipping or billing info. Selecting an existing shipping/billing is on a modal but adding a new shipping/billing required a page reload. Additionally, we required users to type unnecessary information such as retyping their billing address even if it‚Äôs the same as their shipping address.

Typing and Error Checking System

One of the main goals of this project was to make the flow faster. Upon further inspection, checkout time tended to increase when customers fill out their information incorrectly. To solve this, we incorporated an address autocomplete feature, city and state autofill feature (for customers who skip address autocomplete), address recommendation feature, and an error checking system. The new error checking system applied to the rest of the site as well to make it a consistent experience for customers.

Google Autocomplete, Autofill City State, Address Verification, Message System

üîé Usability Testing

To validate our designs, we conducted 12 remote usability testing sessions with potential and existing customers. 6 were conducted on desktop web and 6 were conducted on mobile web. We asked each participant to complete a set of common tasks and scenarios for online shoppers.

After reviewing the testing results with the stakeholders, the users had no major difficulties going through the checkout flow. However, there were a few hiccups and confusions. Thus, we improved the following features based on user feedback:

Autofill City and State

Customers who did not use Google address recommendation preferred filling out the Zip Code first and having it autofill City and State to save time.

Copywriting

While all participants were able to successfully complete the checkout flow, there was some confusion around the copy. Some were confused why the ‚ÄòCard & Other Options‚Äô section led to the shipping section first instead of the billing section. Others were confused why we asked for a phone number in the shipping section. Due to this, we revisited the copy again to make sure all steps were clear.
