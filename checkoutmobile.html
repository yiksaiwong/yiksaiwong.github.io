TOMO Checkout Redesign

(iOS App)



Overview

Scrolling for unique products is easy on Touch of Modern (TOMO), an e-commerce company for modern men. However, customers are not completing their checkout process, leaving us with a cart abandonment rate of 71.06% on our iOS App.

The goal of this project was to redesign the checkout flow to reduce cart abandonment rate and match our new checkout flow on web.

Responsibilities: User Research, Prototyping, Interaction Design, Visual Design, User Testing, Quality Assurance Testing

01. Research & Analysis

üòü Problems with the Current Checkout

Similar to web, it has been a few years since the checkout flow was redesigned. Since then, we‚Äôve continuously added new features to the existing checkout despite various limitations. We analyzed our current flow and discovered the following pain points:

  • Screens have unique, inconsistent layout and styling
  • Lots of back and forth between steps
  • Lacks consideration for all use cases (e.g. Users cannot use express checkout to purchase age-restricted items or select shipping speed)
  • Incomplete typing and error checking system
  • Unlike web, no payment plan options (Sezzle and Affirm) offered

In order to improve the checkout process, we planned to implement the following changes after doing competitive analysis and checkout reassessment:

  • Make the checkout flow faster
  • Remove restrictions for express checkout
  • Add payment plan options

Changing the Overall Layout

Following our solution for web, we decided to also put "Express Checkout" in checkout instead of cart. However, once we implemented it onto mobile app, we realized our solution required customers to continuously click "continue" on sections that they might not necessarily use. To solve this issue, we re-evaluated which steps were truly necessary and created a flow that avoided unnecessary clicks and felt more intuitive. We also updated the web version to be consistent.

Add a Typing and Error Checking System

Similar to web, our mobile app does not have a proper error checking system which causes an increase in checkout time for customers. To solve this, we incorporated an address autocomplete feature, city and state autofill feature (for customers who skip address autocomplete), address verification feature, and an error checking system. The new error checking system will apply to the rest of the app as well to make it a consistent experience for customers.

üîé User Testing

To validate our designs, we conducted 6 remote usability testing sessions with potential and existing customers. Each participant was asked to complete a set of common tasks and scenarios for online shoppers. After reviewing the results with stakeholders, we improved the following based on user feedback:

Not Showing the Next Step

Since we decided not to put numbers in the app to indicate the steps, we found many users who finished filling out "Shipping" would click on "Payment Method", instead of clicking on "Continue". The "Payment Method" section is also grayed out and minimized. Due to this, we got rid of indicating what the next steps will be.

Alert Messages

To avoid misclicks, we decided to add a modal alert box that would trigger if they try to go back to a previous page when they have started filling out a section. This is to avoid the users from having to retype their information again.

Rewrite Copy

Although all participants were able complete the checkout flow, there was still some confusion around copy (similar to web). Again, we revisited the copy again to make sure all steps were clear.
