TOMO Store Filters

Touch of Modern (TOMO) is an e-commerce website catered towards modern men. We offer a large selection of uniquely curated products that we organize into categories, called stores, to make the shopping experience less overwhelming. However, within each store, there are still hundreds of products to filter through. Thus, having a clean and easy to use filter feature is very important for our site. The goal of this project was to update the existing filter to place a larger emphasis on product photos and to revamp the visual design.

01. Research

Challenge

The current store filter layout takes too much attention away from the products and the visual design of the layout needs to be updated.

Goal

Update the store filter to look more clean and polished while ensuring that the products and product photos are the main focus. The solution should be a filter feature that users can easily navigate while still being able to smoothly view and scroll through the list of products.

Audience

New and old customers (most likely males in their 30’s)

02. Analysis

Because this was an unfinished project passed on from another designer, the foundation of the filter had already been set. After reviewing the existing designs and doing some research on my own, I had a better sense of how the filters should work and what needs to be done. Here are the main changes I thought about during the analysis phase:

  • Filter should not take up too much space. When scrolling down, it should collapse.
  • Include more hierarchical categorization.
  • Add a brand filter, giving users the ability to search for brands (can be an infinite amount) and scroll through brands easily.
  • For color filter, limit the number of colors we currently have to ~20.
  • Mobile web version - Displaying all filter categories and their dropdowns on the same page is not ideal since some filters, for example, the brand filter, can have an infinitely long list to scroll through. A solution might be to try leading them to a different screen for each filter.

04. Test & Reflect

A few rounds of QA were done with the engineers to ensure the filter looked and worked the way it should. I also created an excel sheet that tracks any bugs and issues along with potential fixes for the engineers. Additionally, we tested the filter internally to ensure that there were no additional issues.

Lessons learned

  • Always check, test, and iterate. There will always be some edge cases that you missed initially.
  • It is good practice to ask for opinions from different people. You might think your current solution is the best but there might be an even better approach.

Future Plans

  • Add multi-select for categories within each store.
  • Expand the filter feature to other sections of the site, like our sales page.
  • Add a sort-by feature as a category in the filter feature.
